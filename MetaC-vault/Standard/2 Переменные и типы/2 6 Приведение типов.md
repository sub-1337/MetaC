
### Низлежащий тип
Все абстрактные типы во время выполнения конкретизируются
определённым значением.
Наример `int` может стать i32, u64 и т.д.
При этом возможно наличие преобразований типов 
между собой под капотом.


### Неявное приведение

Если переменная A относится к надмножеству переменной B
 приведение  A к B будет приводится автоматически.
В обратную сторону приведение может не всегда работать
и его надо указывать явно.

```
let mut A : int;
let mut B : i32 = 10;

A = B; // тип A i32
```

```
let mut A : int;
let mut B : i32 = 10;

B = A; // Корректность этого будет проверена
```
Вполне возможно компилятор выдаст ошибку, если решит 
что происходит сужающее приведение.

Если мы имеем 2 переменные на одинаковом уровне абстракции
например `i32` и `f32` то для присвоение одной переменной другой
обязательно делать это явно если возможна потеря точности.

### Явное преобразование
```
let a : i64 = 1000;
let b : i32 = cast(a);
```
Автоматический вывод типа и приведение.

```
let a : i32 = 100;
let b : i8 = cast(a, i8);
```
Явное приведение типа. 

### Дробное и целое
Приведение дробного к целому всегда сопровождается типом приведения
`floor`, `ceil`, `round`, `trim`

### Сужение типов
```
let a : i8 = 5;
let b : i16 = a; // ok - расширение диапазона

let a : i16 = 10;
let b : i8 = a; // тоже ok  - число в своих пределах и оно известно

let a : i16 = func();
let b : i8 = a // warning - сужение типа

let a : i16 = 10;
let b : i8  = a.byte[0]; // ok - взятие части байт
let c : i8  = a.byte.any; 
```
