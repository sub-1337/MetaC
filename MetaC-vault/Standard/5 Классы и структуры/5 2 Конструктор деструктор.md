
```
 struct Point 
 {
   public:
     let mut x : real = 0.0;
     let mut y : real = 0.0;
     
	 fn::ctor Point(let const x, let const y)
	 {
		 init this.x = x;
		 init this.y = y;
	 };
	 fn::dtor Point() {};
 }
```

Конструкторы называются именем класса и тип функции
объявлен как `fn::ctor`.
Деструкторы тоже называются именем класса и тип функции
объявлен как `fn::dtor`.

Чтобы явно обратиться к конструктору/деструктору надо указать это
```
Point::ctor::Point(); // Обращение к констурктору
Point::dtor::Point(); // Обращение к деструктору
```

Изнутри класса можно указать `ctor::Point` 

Ключевое слово `init` действующее в пределах конструктора 
обозначает переопределение дефолтного значения 
членов класса на указанное значение. Это аналог списка инициализации
для c++, но более гибкий.

### cast::auto
Ключевое слово в конструкторе которое говорит, что можно автоматически
приводить к типу. (как `explicit` в c++  но наоборот)

```
fn::ctor cast::auto Point(let const val)
{
	init this.val = val;
};
```

Применительно к деструктору эта директива аналог `virtual`
деструктора. Для деструктора если это возможно это флаг по умолчанию.

### cast::explicit
Применительно к  конструктору значит, что автоматически создавать 
объект можно только явно. Вариант для конструкторов по умолчанию.
В c++ по умолчанию все деструкторы `explicit`.
В meta C это не так, но мы можем использовать директиву
`cast::explicit` для того, чтобы деструктор не был виртуальным.
